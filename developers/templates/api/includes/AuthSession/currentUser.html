Returns information about the currently authenticated user's session. If the caller is not authenticated they will get a 401 Unauthorized status code.

Note that it is generally preferrable to use HTTP BASIC authentication with the REST API. However, this resource may be used to mimic the behaviour of SYMPHONY's log-in page (e.g. to display log-in errors to a user).

Responses
  • Status 200
    Returns information about the user's session if the user is authenticated.

  • Status 400
    Bad Request - Request does not have a valid format, all required parameters, etc. 

    {
  "response": {
    "msg": "Invalid Request..Authorization Token required",
    "success": false,
    "result": null
  },
  "statusCode": 400
}
  • Status 401
    Unauthorized Access - No currently valid authorization has been made.
  • Status 403
    Forbidden Access - Access to this service or resource is forbidden with the given authorization.
  • Status 404
    Not Found - Service or resource was not found
  • Status 500
    System Error - Specific reason is included in the error message
